# GitHub Issue Reporter - 詳細リファレンス

## 概要

このスキルは、Claude Codeのすべての重要なアクションをGitHub Issue #5（Private Repository）に自動報告します。
従来のプロンプト方式では報告漏れが頻発していたため、スキルとして強制ルール化されました。

## 報告が必要な全シナリオ

### 1. タスク完了時（Completion）
作業が正常に完了した場合。

**報告内容:**
- 完了した作業の詳細
- 変更されたファイル一覧
- コミット情報（該当する場合）
- デプロイ情報（該当する場合）

### 2. 質問・確認時（Question）
ユーザーに質問や確認を求める場合。

**報告内容:**
- 質問の内容
- なぜその情報が必要か
- 現在の作業状況
- 想定される選択肢

### 3. エラー発生時（Error）
処理中にエラーが発生した場合。

**報告内容:**
- エラーの詳細
- エラーが発生した箇所
- 試みた解決策
- 推奨される次のアクション

### 4. 待機時（Waiting）
追加の情報や入力を待つ場合。

**報告内容:**
- 何を待っているか
- なぜ待機が必要か
- 期待される入力

### 5. 情報提供時（Information）
分析結果や調査結果を報告する場合。

**報告内容:**
- 調査・分析の結果
- 発見事項
- 推奨事項

## 報告フォーマット

### マークダウン形式（必須）

報告は必ずマークダウン形式で記述する:

```markdown
## タイトル

### 実施内容
- 項目1
- 項目2

### 変更ファイル
- `file1.py` - 説明
- `file2.js` - 説明

### 結果
[結果の詳細]
```

## 禁止事項

### 絶対禁止:
1. **報告の省略** - いかなる理由でも報告を省略しない
2. **報告の延期** - 「後でまとめて報告」は禁止
3. **報告の簡略化** - 必要な情報を省かない
4. **質問時の報告スキップ** - 質問だからといって報告を省略しない
5. **エラー時の報告スキップ** - エラーこそ報告が必要

## トラブルシューティング

### 報告が失敗する場合

1. **ネットワークエラー**: 再試行する
2. **認証エラー**: GITHUB_TOKEN を確認
3. **タイムアウト**: 報告内容を短くして再試行

### 報告漏れを防ぐための自己チェック

回答を生成する前に自問:
1. この回答には報告すべき内容があるか？ → あれば報告
2. ユーザーに何かを求めているか？ → あれば報告
3. エラーが発生したか？ → あれば報告
4. 作業が完了したか？ → あれば報告

**上記のいずれかに該当すれば、必ず報告コマンドを実行する**
